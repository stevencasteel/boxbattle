1. RUNTIME STACK AND CONFIGURATION BOUNDARY
───────────────────────────────────────────────────────────────────────────────

Primary application stack:
- Presentation: React 19, TypeScript 6.0, Vite 8.0, Zustand 5.0
- Simulation: 60Hz accumulator with custom swept collision solver
- Visuals: Procedural HTML5 Canvas rendering (No external binary static textures)
- Audio: Native Web Audio API procedural synthesis with limiter protection
- Auxiliary: canvas-confetti, lucide-react

The repository includes standard properties in `src/core/Units.ts` governing player kinematics, basic boss scales, collision margins, and audio spatialization max ratios.

2. BOOT, FIRST-GESTURE, AND APP INITIALIZATION
───────────────────────────────────────────────────────────────────────────────

Relevant files:
- `src/main.tsx`
- `src/App.tsx`
- `src/hooks/useBootSequence.ts`
- `src/hooks/useFirstGesture.ts`

App initialization runs inside `main.tsx` which invokes `GauntletDirector.init()` to register global keyboard listeners prior to mounting. Upon gesture click/touch, the audio graph initializes, and the background music sequencer fires.

3. GAUNTLET, ENTITY, AND VISUAL PROFILE ARCHITECTURE
───────────────────────────────────────────────────────────────────────────────

Relevant files:
- `src/entities/BaseEntity.ts`
- `src/entities/BaseMinion.ts`
- `src/entities/Player.ts`
- `src/entities/Boss.ts`
- `src/core/visuals/ShapeRenderer.ts`
- `src/core/design/ColorRoles.ts`

Entities utilize a flat composite object architecture with custom logical components (Physics, Health, Dash, Melee, Fireball, Heal, InputReceiver). Instead of standard software 3D box models, all entities expose a procedural `VisualProfile` defining their shape family (perfect-square, hex, diamond, kite, needle, saw, blister, cage) and color role.

Color Roles:
- player-agency: Green HSL (Pure green reserved only for player actions)
- boss-lethal: Red/Dark Red HSL (Hostile boss actions)
- telegraph: Yellow HSL (Warning states before attack commitment)
- determination: Violet HSL (Healing, pogo helixes, recovery barks)
- minion-logic: Cyan HSL (Machine actions, tracking indicators)
- minion-organic: Khaki/Brown HSL (Contaminated grow lanes)
- arena-stone: Charcoal HSL (Static background solid bodies)

The Player is maintained as the *only perfect square* in the simulation world, preserving visual priority.

───────────────────────────────────────────────────────────────────────────────
4. MAIN LOOP, FIXED-STEP SIMULATION, AND STATE PROJECTION
───────────────────────────────────────────────────────────────────────────────

Relevant files:
- `src/core/GameLoop.ts`
- `src/core/Engine.ts`
- `src/core/StateProjectionSystem.ts`

The game loop operates on requestAnimationFrame (rAF) but advanced state integrations are capped at a strict 60Hz accumulator loop. At each frame tick, previous position matrices are cached, sub-stepping is processed via dt increments, and the StateProjectionSystem replicates player health, boss health, healing charges, and determination counters into the Zustand store for reactive UI panels.

5. PHYSICS WORLD, BROAD-PHASE, AND DYNAMIC CACHE REBUILDS
───────────────────────────────────────────────────────────────────────────────

Relevant files:
- `src/core/PhysicsWorld.ts`
- `src/core/StaticMapRenderer.ts`
- `src/core/WorldRenderer.ts`

Solid, platform, and hazard boundaries are partitioned inside cell buckets utilizing a spatial hashing grid with 200px cell width margins. 

The physics world exposes a dynamic `rebuild(solids, hazards, oneways)` method. Whenever a dynamic dissolve platform collapses or respawns, the active solid geometries are filtered on the fly, `PhysicsWorld` clears its grid buckets and re-indexes them, and the `WorldRenderer` invalidates the static map cache via a public `resetCache()` delegator, redrawing neon boundaries for the new layout.

───────────────────────────────────────────────────────────────────────────────
6. INPUT MAPPING AND GLOBAL KEYBOARD WARPS
───────────────────────────────────────────────────────────────────────────────

Relevant files:
- `src/core/InputProvider.ts`
- `src/core/GauntletDirector.ts`

In addition to custom keyboard rebindings (Preset 1, Preset 2, and Custom) and virtual joysticks for mobile screens, the `GauntletDirector` registers a global keyboard handler. Pressing digit keys '1' through '7' at any time instantly triggers a warp sequence, resets the gameplay store, transitions screen routing, re-indexes the physics world, and spawns the player at the start of that respective stage layout.

7. PLAYER TRAVERSAL, COMBO SCALE CHAINING, AND COMBAT MECHANICS
───────────────────────────────────────────────────────────────────────────────

Relevant files:
- `src/entities/handlers/PlayerInputHandler.ts`
- `src/entities/handlers/PlayerCombatHandler.ts`
- `src/core/audio/sfx/BossSFX.ts`

The player controller processes ground movement, coyote jump buffers, wall sliding, and dashes. Downward sfx strikes (pogoes) and hit confirms are dynamically pitched up according to 12-Tone Equal Temperament (12-TET) intervals based on C Dorian pitch multipliers. The active combo count scales pitches progressively, resetting immediately on damage or landing.

───────────────────────────────────────────────────────────────────────────────
8. TRAVERSAL HAZARDS & PATH OF PAIN (PoP) STATE MACHINES
───────────────────────────────────────────────────────────────────────────────

Relevant files:
- `src/core/systems/TraversalHazards.ts`

To support advanced platforming trials, three traversal state machines operate in the fixed-step physics integration loop:
- DissolvePlatform: contact starts a 0.45s warning timer where the platform cracks in yellow, vanishes for 1.6s, then respawns as a desaturated dashed outline for 0.5s before solidifying.
- PogoPost: safe only from downward melee attacks, which triggers a high-impulse rebound, and refreshes player double-jump and dash charges. Landing on it from any other angle triggers damage and knockback.
- DashResetGate: a static green diamond outline that instantly replenishes player dash and double-jump charges upon overlap, entering a 2.0s cooldown.

9. ROSTER V2, SPAWN CAGES, AND COMBAT AI
───────────────────────────────────────────────────────────────────────────────

Relevant files:
- `src/entities/NewGauntletMinions.ts`
- `src/entities/MinionFactory.ts`
- `src/core/visuals/MinionVisuals.ts`

Minions utilize distinct geometric visual profiles:
- Turret Seed: Hexagonal logic core firing tracking bullets.
- Pit Lancer: Horizontal pusher trying to shove the player into hazards.
- Compass Wasp: Small diamond wasp orbiting and diving along tangent lines.
- Clampjaw: Lumpy heavy box moving wall-to-wall with high hp.
- Hymn Nail: Vertical needle pogo anchor that turns hostile if overused.
- Blister Ox: Massive organic blob releasing shockwaves on landing.
- Bell Hammer: Heavy slam hammer targeting marked tiles.

Spawn Cages draw custom wireframes previewing the incoming silhouette (e.g. tuning forks for Hymn Nail, unfolding hexagons for Turret Seed, membrane bubbles for Blister Ox) during spawn initialization.

───────────────────────────────────────────────────────────────────────────────
10. DYNAMIC CANVAS BOSS PORTRAITS AND EVENT DIALOGUE CONSOLE
─────────────────────────────────────────────────────────────────────────

Relevant files:
- `src/components/DialogueConsole.tsx`
- `src/core/BattleDirector.ts`

The DialogueConsole embeds real-time `<canvas>` portrait renderers drawing mathematical identity patterns (one diagonal crack for Prime Wound, vertical prison bars for Scarlet Lock, circular cuts for Carminal Orbit, teeth edges for Vermilion Needle) instead of static images. The BattleDirector coordinates stage-specific narrative event barks based on the active gauntlet index, tracking custom phase thresholds and death sequences.
